# 贡献指南

感谢你对 Vibe Coding Collection 的关注！我们欢迎社区贡献。

## 🤝 如何贡献

### 报告问题
如果你发现了错误或有改进建议：
1. 在 [Issues](../../issues) 页面检查是否已存在相似问题
2. 如果没有，创建新的 Issue，包含：
   - 清晰的标题
   - 详细的描述
   - 复现步骤（如果是 bug）
   - 期望的行为
   - 截图（如果适用）

### 提交新资源
如果你想添加新的教程、文档、GitHub 仓库或新闻：

1. **Fork 本仓库**
2. **创建分支** (`git checkout -b feature/add-resource`)
3. **编辑相应文件**:
   - 教程 → `docs/tutorials.md`
   - GitHub 仓库 → `docs/github-repositories.md`
   - 新闻 → `docs/news.md`
   - 其他资源 → `README.md`
4. **遵循现有格式**
5. **提交更改** (`git commit -m 'Add: XXX 资源'`)
6. **推送到你的 Fork** (`git push origin feature/add-resource`)
7. **创建 Pull Request**

### 贡献内容类型

#### ✅ 欢迎的内容
- 高质量的教程链接
- 实战项目案例
- 有价值的 GitHub 仓库
- 官方新闻和更新
- 技术深度文章
- 翻译和本地化内容
- 最佳实践总结

#### ❌ 不接受的内容
- 纯广告性质的内容
- 低质量或过时的资源
- 未经证实的传闻
- 侵犯版权的内容
- 与 Vibe Coding 无关的内容

### 资源质量标准

提交资源时，请确保：

1. **准确性** ⭐⭐⭐⭐⭐
   - 信息经过验证
   - 技术细节正确
   - 链接有效可访问

2. **可靠性** ⭐⭐⭐⭐⭐
   - 来源权威可信
   - 内容持续更新
   - 维护活跃

3. **完整性** ⭐⭐⭐⭐
   - 信息全面
   - 有清晰的结构
   - 包含必要细节

4. **权威性** ⭐⭐⭐⭐
   - 官方资源优先
   - 知名社区认可
   - 专家推荐

## 📝 提交规范

### Commit Message 格式
使用清晰的 commit message：

```
<type>: <subject>

<body>

<footer>
```

**Type 类型**:
- `Add`: 添加新资源
- `Update`: 更新现有内容
- `Fix`: 修复错误
- `Docs`: 文档改进
- `Style`: 格式调整
- `Refactor`: 重构
- `Test`: 添加测试

**示例**:
```
Add: Claude Code 官方教程链接

- 添加了 Anthropic 官方的 Claude Code 教程
- 包含安装、配置、使用说明
- 适合初学者

Closes #123
```

### Pull Request 标题
PR 标题应该清晰描述更改：

```
[类型] 简短描述

例如：
[Add] 新增 Cursor 编辑器进阶教程
[Update] 更新 GitHub 仓库列表
[Fix] 修复 README 中的链接错误
```

## 🌍 国际化

我们欢迎多语言内容！

### 添加中文内容
- 添加到 `docs/` 目录
- 使用简体中文
- 保持与英文版本结构一致

### 添加其他语言
1. 创建 `docs/<lang>/` 目录
2. 翻译或创作内容
3. 在 README 添加语言链接
